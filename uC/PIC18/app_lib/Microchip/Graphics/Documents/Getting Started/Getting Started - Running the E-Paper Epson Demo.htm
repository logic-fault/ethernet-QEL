Running the Demo:

This demo project is created to run on supported Microchip and Epson demo boards and development hardware. First program the PIC with the demo firmware, precompiled hex files are available in the "Precompiled Hex” sub-directory found in the demo directory.

For proper demo execution, the ExternalResource-E-Paper.hex file in "Resources” folder must also be programmed into external flash SPI memory installed on the PICtail board. The PIC firmware should automatically prompts for this file if no valid external memory data was found.

All programming are initiated through the PC utility - memory_programmer.jar. You can find this utility in the following Microchip Application Libraries path:

.<install_dir>\Microchip\Graphics\bin\memory_programmer

Refer to the memory_programmer help file for details on the programmer utility operation.

 

After reset, the demo will cycle through all of its screens with no user interaction.  Pressing one of the three buttons on the PICtail board will stop this.  The left and right buttons on the PICtail board can be used to manually switch the demo screens.

·         Title Screen – This screen tells the user a little about the demo itself.  It shows the Epson and Microchip logo along with the controller name S1D13522. After two seconds, the auto demo will start if any one of the three buttons are not pressed

·         Fonts – This screen displays six strings of the same type of font at different sizes.  The first three (the largest ones) use anti-aliasing.

o     S1 – Switch to the EKG screen

o     S2 – does nothing

o     S3 – Switch to Foreign Fonts screen

·         Foreign Fonts – This screen displays the phrase “Hello World” in different languages

o     S1 – switch to Fonts screen

o    S2 - does nothing

o    S3 – switch to the Picture screen

·         Picture – Displays a picture.  The picture is rendered using the same routine call, PutImage, as any other graphics images.

o    S1 – switch to Foreign Fonts screen

o    S2 – change the picture, there are three total

o    S3 – switch to Widgets screen

·         Widgets – Displays GOL, Graphics Object Layer, widgets.

o    S1 – switch to Picture Screen

o    S2 – does nothing

o    S3 – switch to Chart screen

·         Chart – Displays the chart widget as a bar graph.

o    S1 – switch to Widgets screen

o    S2 – does nothing

o    S3 – switch to QR screen

·         QR – This screen displays QR codes which can be decoded. The first, smaller, QR code is www.epson.com  The second, larger, QR code is www.microchip.com/graphics

o    S1 – switch to Chart screen

o    S2 – changes the QR code

o    S3 – switch to the EKG screen

·         EKG – Shows a fast update of the E Ink display

o    S1 – switch to the QR screen

o    S2 – does nothing

o    S3 – switch to the PIP screen

·         PIP – Shows the S1D13522 hardware picture-in-picture and cursor overlay capabilities. The key highlight here is, when the overlay layers are removed, the background image is retained and does not need to be redrawn.

o    S1 – switch to the EKG screen (when PIP and cursor are not displayed)

o    S2 – Follow on screen instruction (once PIP screen is shown, must choose 'Exit' to be able to use S1/S3 to navigate to other demo screens)

o    S3 – switch to the Font screen (when PIP and cursor are not displayed)

Converting Images for Display on the Epson S1D13522 Controller

The Epson S1D13522 controller is used for displaying graphical data on E Ink display panels. An application using this controller can display bitmap images. These Bitmap images can be in 4 and 1 bit per pixel formats.  The controller does not use the palette from the bitmap image, as it has an internal fixed palette. To maximize image clarity, the bitmap image should be converted to the fixed palette grayscale. This can be done by using a program like GNU Image Manipulation Program (www.gimp.org).  The Microchip Graphics Resource Converter (GRC) is then used to convert the bitmap images into PIC internal flash or external memory resources.  The conversion of bitmap images does require for a non-default option in the Graphics Resource Converter to be selected.

GIMP: Converting Bitmaps to Fixed Grayscale Palette

Please refer to the Graphics Library Help file, which is located at <install_dir>/Microchip/Help, section Miscellaneous Topics > Converting Images to Use a Common Palette in GIMP for detailed information on this topic. The palette files used are located in <install_dir>/Microchip/Graphics/palettes. Where install_dir is the location of the Microchip Application Libraries installation.

Converting Images using Graphics Resource Converter

When converting bitmap images using the Graphics Resource Converter (GRC), the ‘no line padding’ option must be enabled to insure proper pixel alignment when using with the S1D13522 driver/controller. The S1D13522 driver expects images not to have any horizontal line padding. The default of the GRC is to provide horizontal line padding on byte boundaries.

 

To enable the ‘no horizontal line padding’:

    Under the settings dialog box, select the “No Line Padding” in the “Bitmap Image Options” group box.

 

 

 

 

 

Having this option checked is a requirement for using images with the S1D13522. In the GraphicsConfig header file, the macro, USE_BITMAP_NO_PADDING_LINE, this will allow the compile time check produced by the GRC to pass.

  

 

Configuring the Software

The demo can be run using the E-Paper PICTail and the Explorer 16 Development board This is set in the project HardwareProfile.h file by adding the corresponding Graphics hardware profile.

 

·         For 16-bit PMP interface:

#include "Configs/HWP_EPD_EX16_16PMP_QVGAv1.h”

·         For 8-bit PMP interface:

#include "Configs/HWP_EPD_EX16_8PMP_QVGAv1.h”

·         For SPI interface:

#include "Configs/HWP_EPD_EX16_SPI_QVGAv1.h”

 

The hardware profile file name refers to the combination of Development Boards that are used to run the demo. Refer to the abbreviations summary (link) for details.

Note 1:

E-paper Display PICtail Plus Board jumper settings:

DIP Switch (SW1) is for selecting the host interface mode.

For 16-bit parallel host interfaces (default)

SW1-1 = on,

SW1-2 = on,

SW1-3 = don’t care,

SW1-4 = off,

SW1-5 = off

For 8-bit parallel host interfaces

SW1-1 = off,

SW1-2 = on,

SW1-3 = don’t care,

For serial – SPI1 host interfaces

SW1-1 = off,

SW1-2 = off,

SW1-3 = on,

 

DIP Switch (SW2) is for SPI chip select and channel selection of the 64-Megabit flash memory.

For SPI channel 2 and ALT_FLASH_CS (default)

SW2-1 = off,

SW2-2 = off,

SW2-3 = off,

SW2-4 = on

Jumper (J3) is for production calibration.

For normal operation (default)

J3 pins 1-2 jumped

For detailed jumpe settings information, refer to the PICtail info sheet that comes with the E-paper Display PICtail Plus Board.

Note 2:

PIC24FJ256GB210 PIM jumper settings:

16-bit parallel interface mode - JP1, JP2, JP3 'P', JP4 - open

8-bit parallel or serial host interface mode - JP1, JP2, JP3 'U', JP4 - close
